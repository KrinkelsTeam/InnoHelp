[Setup]: UseSetupLdr


Допустимые значения:
yes или no

Значение по умолчанию:
yes

Описание:

Эта директива сообщает компилятору о типе создаваемой программы установки. Если указано значение yes, то все данные установки компилируются в один файл EXE. При значении no все данные установки компилируются как минимум в три файла: setup.exe, setup-0.bin и setup-1.bin. Возможная причина, по которой следует использовать значение no, является отладка программы установки.

Замечание: Не используйте UseSetupLdr=no в установках с разделением данных (DiskSpanning=yes). Когда параметр UseSetupLdr имеет значение yes, программа установки копируется и запускается из пользовательского каталога TEMP. Этого не происходит, если параметр UseSetupLdr имеет значение no, и может привести к ошибкам, когда Windows попытается найти файл setup.exe там, где его нет (например, в приводе находится другой диск).

Замечание: Не используйте UseSetupLdr=no на больших инсталляторах. Это приведёт к задержкам во время запуска при проверке цифровой подписи. Вместо этого используйте разделение на диски. Дополнительную информацию смотрите в разделе SignTool. Также цифровая подпись программы установки с UseSetupLdr=no приведёт к недействительной цифровой подписи программы удаления.
